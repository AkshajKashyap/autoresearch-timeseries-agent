# Limitations

## Synthetic Data

The synthetic datasets are deterministic and useful for controlled development, but they
are not a substitute for real-world benchmarks. They can make some methods look better
or worse depending on how closely their assumptions match the generated dynamics.

## CSV Demo

`data/raw/example_timeseries.csv` is a local ingestion demo generated by
`scripts/create_example_csv_dataset.py`. It proves the pipeline can read local tabular
time-series data, but it is not an external benchmark or a claim about real-world
forecasting performance.

## Blocked Shuffle

`blocked_shuffle` is diagnostic-only. It can show whether a model has capacity when
temporal distribution shift is reduced, but it is not a realistic forecasting split.

## External Benchmarks

No real external benchmark dataset is included yet. Any future benchmark should document
data provenance, licensing, frequency, missing-value behavior, target definition, and
evaluation assumptions.

## Production Deployment

There is no production serving layer, web app, API server, Docker setup, or monitoring
system. The project is currently a local experiment and reporting foundation.

## Agent Scope

The current agent is deterministic and config-bounded. It does not call OpenAI,
LangChain, LangGraph, external APIs, or write source code.
